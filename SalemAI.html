الجدول الكامل لمراحل المشروع

المرحلة المسؤول المهام المطلوبة المدخلات المخرجات
1) تحليل المتطلبات (Requirement Analysis) فريق كامل (مهندس + أطباء) تحديد السلوكيات المطلوبة – قائمة التشخيص – شكل التقرير النهائي الفكرة العامة + خبرتكم الطبية وثيقة متطلبات واضحة + قائمة السلوكيات المطلوب تحليلها
2) إعداد سياسة الخصوصية والموافقة القانونية المستشار القانوني صياغة موافقة ولي الأمر – سياسة استخدام الفيديوهات – مدة الاحتفاظ نوع البيانات – آلية الرفع – أعمار الأطفال نموذج موافقة رسمي + سياسة خصوصية مكتوبة
3) تصميم النظام (System Architecture) مهندس ذكاء اصطناعي + باكند تحديد تدفّق البيانات – تحديد الأدوات – اختيار النموذج – رسم البنية وثيقة المتطلبات مخطط معماري للنظام + خطة الأدوات والنماذج
4) إعداد وتجهيز البيانات (Data Preparation) الأطباء + مهندس ML جمع فيديوهات أولية – قص المقاطع – تجهيز ملفات الصوت فيديوهات الجلسات – موافقات أولياء الأمور مجموعة بيانات نظيفة جاهزة للوسم
5) وسم البيانات (Data Annotation) الأطباء وسم السلوكيات: تواصل بصري – حركات متكررة – ارتجاف – استجابة للمحفز مجموعة فيديوهات التدريب ملف وسم كامل بصيغة CSV/JSON
6) استخراج الميزات (Feature Extraction) مهندس ML استخراج وضعية الجسم (MediaPipe) – استخراج نصوص الصوت (Whisper) – تحويلها لأرقام الفيديوهات الموسومة Features جاهزة للتدريب (keypoints + صوت + مؤشرات)
7) تدريب النموذج الأولي (Model Training) مهندس ML اختيار نموذج بسيط (LightGBM/Transformer صغير) – تقسيم البيانات – معايرة النتائج البيانات المستخرجة + الوسوم نموذج يعطي احتمالات (Autism, ADHD, Anxiety, Depression, Behavioral issues)
8) اختبار النموذج وتقييمه الأطباء + مهندس ML مقارنة نتائج النموذج مع الرأي الطبي – تعديل العتبات – تحسين الأداء مخرجات النموذج + عينات جديدة تقرير دقة النموذج + الإصدار الأول المستقر
9) بناء واجهة رفع الفيديو (Front-End) مطور الواجهة تصميم صفحة رفع فيديو – عرض progress – تصميم شاشة التقرير تصميم مبدئي – API جاهز واجهة تعمل لرفع الملفات
10) بناء API وتحليل الفيديو (Backend) مهندس باكند API لاستقبال الفيديو – تمريره للنموذج – معالجة النتائج – حذف الملفات بعد المدة النموذج النهائي + مخطط البنية API جاهز للإنتاج
11) توليد التقارير (Report Generation) مهندس باكند + مطور الواجهة إنشاء ملف PDF – إضافة الرسوم البيانية – كتابة النتائج بنسب – توصيات علاجية نتائج التحليل + تصميم التقرير + توصيات الأطباء تقرير PDF كامل لكل جلسة
12) الإطلاق الداخلي (Internal Launch) الفريق كامل تجربة النظام مع 5–10 جلسات حقيقية – إصلاح الأخطاء – تحسين الواجهة نسخة شبه نهائية نسخة MVP مستقرة
13) الإطلاق الرسمي للمراكز (Production Release) الفريق كامل تشغيل الخادم – تفعيل الحسابات – مراقبة الأداء نسخة MVP المنتج النهائي المستخدم في المراكز

الفرد المسؤوليات
مهندس الذكاء الاصطناعي (ML) معالجة الفيديو – استخراج الميزات – تدريب النموذج – ضبط الدقة – دمج الصوت والفيديو
مطور الباكند بناء API – إدارة التخزين – الحذف المؤقت – الأمن – توليد التقارير
مطور الواجهة (Front-End) واجهة رفع الفيديو – واجهة عرض التقارير – تصميم تجربة الاستخدام
الأطباء النفسيون تعريف السلوكيات – وسم البيانات – مراجعة نتائج النموذج – صياغة التوصيات
المستشار القانوني صياغة الموافقات – سياسة حماية بيانات الأطفال – شروط الاستخدام
